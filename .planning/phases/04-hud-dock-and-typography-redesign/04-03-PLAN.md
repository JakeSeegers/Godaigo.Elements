# 04-03: Bottom Dock — Stone Chips + Player Roster Cards

## Objective

Add a slim persistent bottom dock (68px) that shows: (1) the shared source pool for each
element as compact colored chips (`E 18` format), and (2) small player roster cards showing
each player's name, color dot, and AP. This replaces the current `.action-bar` (Scrolls, Cast
Spell, Undo) which is repurposed into the left panel so the dock can use the full bottom.

---

## Context

**Current bottom area** (index.html ~line 425–443):
The `.action-bar` sits in grid-area `actions` and contains: Scrolls button, Cast Spell button,
action-divider, Undo Step button, Leave button. Height: `--action-bar-height: 56px`.

**Current stone data display:** Stone counts live in `.stone-grid` inside the left panel
(`.left-panel`). Each element has:
- `#new-{element}-count` → player pool "0/5"
- `#new-{element}-source` → source pool integer "20"

The dock will display **source pool** values only (shared pool, not per-player), which matches
the v3 mockup intent. The left-panel stone cards retain the per-player pool display.

**Player cards:** The right panel (`#new-opponent-cards`) is populated dynamically by game-ui.js
(~line 746 onward). We'll mirror a compact version of this into the dock.

**Grid layout** (css/styles.css ~line 131–143):
```css
.game-layout {
    grid-template-rows: var(--hud-height) auto 1fr var(--action-bar-height);
    grid-template-areas:
        "hud    hud    hud"
        "status status status"
        "left   board  right"
        "actions actions actions";
}
```

**Current action buttons** that must be preserved: `#scroll-inventory`, `#cast-spell`,
`#undo-move`, `#leave-game`.

**Step A — Add `updateDockStones()` helper function** near other update helpers:

```js
function updateDockStones() {
    const elements = ['earth', 'water', 'fire', 'wind', 'void'];
    elements.forEach(el => {
        const el2 = el === 'wind' ? 'wind' : el; // wind is just wind
        const sourceEl = document.getElementById('dock-' + el + '-count');
        if (!sourceEl) return;
        // sourcePool is window.stonePools from config.js
        const pool = window.stonePools ?? sourcePool ?? {};
        sourceEl.textContent = pool[el] ?? '--';
    });
}
```

**Step B — Call `updateDockStones()` at every point where source pool counts are updated.**

Find all calls to `updateStoneCount(element)` function (lines ~1220, ~2399, ~2516, ~2666).
After each such call (or in the `updateStoneCount` function body itself), add:
```js
updateDockStones();
```

The simplest reliable approach: add `updateDockStones()` at the END of the existing
`updateStoneCount` function body (if one exists) so it runs automatically for every element
update. If `updateStoneCount` is defined in the file, find it and add the call at its end.

**Step C — Add `updateDockPlayers()` function** to populate dock player roster cards:

```js
function updateDockPlayers() {
    const container = document.getElementById('dock-players');
    if (!container) return;
    container.innerHTML = '';
    const pList = window.players ?? players ?? [];
    pList.forEach((player, i) => {
        if (!player) return;
        const card = document.createElement('div');
        card.className = 'dock-player-card' + (i === currentPlayer ? ' active-player' : '');
        const color = player.color || '#888';
        const ap = (i === currentPlayer) ? (window.currentAP ?? currentAP ?? '?') : '?';
        card.innerHTML = `
            <span class="dock-player-dot" style="background:${color};"></span>
            <span class="dock-player-name">${player.name || 'Player ' + (i+1)}</span>
            <span class="dock-player-ap">${ap}AP</span>
        `;
        container.appendChild(card);
    });
}
```

**Step D — Call `updateDockPlayers()` at turn start and after player join.**

Find the section that updates `#hud-player-name` and `#hud-player-dot` at turn start

```js
updateDockStones();
```

Also call both in the `updateOpponentCards()` / opponent card refresh function if one exists,
