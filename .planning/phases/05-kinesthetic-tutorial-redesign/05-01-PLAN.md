```js
placedStones.push({
    id: stoneId,
    x: x,
    y: y,
    type: type,
    element: stoneGroup

```
Immediately AFTER that push (and before the existing `if (typeof spellSystem !== 'undefined' && spellSystem.scrollEffects)` block that handles burningMotivation), insert:
```js
// Tutorial hook — fires after stone added to placedStones
if (window.isTutorialMode && window.TutorialMode?.onStonePlaced) {
    window.TutorialMode.onStonePlaced(type, x, y);
}
```

In `attemptBreakStone(stoneId)`, locate the splice that removes the stone from placedStones:
```js
const index = placedStones.findIndex(s => s.id === stoneId);
if (index !== -1) {
    placedStones.splice(index, 1);
}
```
Immediately AFTER the closing `}` of that if-block (and before the existing `returnStoneToPool(stone.type)` call), insert:
```js
// Tutorial hook — fires after stone confirmed removed
if (window.isTutorialMode && window.TutorialMode?.onStoneBroken) {
    window.TutorialMode.onStoneBroken(stone.type, stone.x, stone.y);
}
```
Note: `stone` is still in scope because it was captured with `const stone = placedStones.find(s => s.id === stoneId)` at the top of `attemptBreakStone`. `stone.type`, `stone.x`, `stone.y` are safe to read even after splicing because `stone` is a reference to the object, not the array slot.
