## Task

Add a source pool guard in `applyScrollEffects()` (or wherever `spellSystem.activated` is incremented) so that casting a scroll when the element's source pool is depleted does NOT count as a win-condition activation.

## Files

- `js/game-core.js` — `applyScrollEffects()` or `spellSystem` activation logic

## Action

1. Read `js/game-core.js` and locate where `spellSystem.activated` is updated after a scroll cast
2. Find the source pool tracking (likely `stoneCounts[element]` or `sourcePool[element]`)
3. Wrap the activation increment with a guard:

```js
// Only count toward win condition if source pool has stones remaining
if (sourcePool[element] > 0) {
    spellSystem.activated[element] = true;
} else {
    // Show message — scroll effect fires but win condition not met
    showStatusMessage(`The ${element} shrine's source is depleted — scroll cast but win condition not met.`);
}
```

4. Confirm `showStatusMessage` (or equivalent) is the correct function to surface this to the player
5. Ensure the scroll EFFECT still fires (stones move, board changes) — only the win-condition check is blocked

## Verify

- [ ] `spellSystem.activated[element]` is NOT set when source pool for that element is 0
- [ ] A visible status message appears when this case is hit
- [ ] Scroll effects (board/stone changes) still fire normally
- [ ] Win condition check still fires and works correctly when pool IS non-zero
- [ ] No JS errors

## Done when

Manually depleting an element's source pool then casting its scroll shows a message and does not trigger a win.
